  <behavior>
    PgTypeMapper.toSchemaType(pgType: String, isPrimaryKey: Boolean = false, enumOptions: List<String>? = null) -> SchemaType:
      - "text"/"varchar"/"char"/"citext" (case-insensitive, strip length suffix `(255)`) → TEXT
      - "int2"/"int4"/"int8"/"int"/"integer"/"bigint"/"smallint"/"numeric"/"decimal"/"real"/"float4"/"float8"/"double precision"/"money" → NUMBER
      - "bool"/"boolean" → CHECKBOX
      - "date" → DATE
      - "timestamp"/"timestamptz"/"timestamp with time zone"/"timestamp without time zone"/"time"/"timetz" → DATETIME
      - "uuid" + isPrimaryKey=true → ID; "uuid" + isPrimaryKey=false → TEXT
      - non-null `enumOptions` (i.e. caller already detected pg user-defined enum) → SELECT
      - "jsonb"/"json" → OBJECT
      - any type starting with `_` (Postgres array type) OR ending with `[]` → OBJECT
      - "bytea" → OBJECT
      - "geometry"/"geography" → LOCATION
      - anything else → OBJECT (fallback)
      - Email-named columns are NOT auto-upgraded to EMAIL here — that is an LLM/heuristic decision in 03-04, not the type mapper's concern.

    SchemaHasher.compute(tableName: String, columns: List<ColumnSchema>) -> String:
      - Build canonical JSON: `{"t":"<tableName>","c":[{"n":"<col>","p":"<pgType>","u":<nullable>},...]}`
      - Columns sorted alphabetically by name BEFORE serialization (column-order independence).
      - Hash SHA-256 over UTF-8 bytes; return lowercase hex (64 chars).
      - Use Jackson ObjectMapper with deterministic key ordering + `MapperFeature.SORT_PROPERTIES_ALPHABETICALLY` to canonicalize.

      - Every named test in 03-00's SchemaHashTest (5 tests) becomes a real assertion against SchemaHasher.compute.
      - Use parameterised tests where it shortens the file.
  </behavior>
