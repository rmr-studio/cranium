  <name>Task 1: Implement ContentValidationService with Tika MIME detection and SVG sanitization</name>
  <files>
    src/main/kotlin/riven/core/service/storage/ContentValidationService.kt,
    src/test/kotlin/riven/core/service/storage/ContentValidationServiceTest.kt
  </files>
  <behavior>
    - detectContentType: Given a JPEG byte stream, returns "image/jpeg" regardless of filename extension
    - detectContentType: Given a PNG byte stream with .txt extension, returns "image/png" (not "text/plain")
    - validateContentType: Given StorageDomain.AVATAR and "image/png", does not throw
    - validateContentType: Given StorageDomain.AVATAR and "application/pdf", throws ContentTypeNotAllowedException
    - validateFileSize: Given StorageDomain.AVATAR and 1MB, does not throw
    - validateFileSize: Given StorageDomain.AVATAR and 3MB, throws FileSizeLimitExceededException
    - sanitizeSvg: Given SVG with script tag, returns SVG without script tag
    - sanitizeSvg: Given SVG with onload event handler, returns SVG without event handler
    - generateStorageKey: Returns format "{workspaceId}/{domain}/{uuid}.{ext}" where uuid is random and ext is derived from detected MIME type
  </behavior>
  <action>
    Create `ContentValidationService` as a `@Service` with constructor-injected `KLogger`:

    1. **MIME detection** using `org.apache.tika.Tika` instance (thread-safe, create once):
       - `fun detectContentType(inputStream: InputStream, filename: String?): String`
       - Use `Tika().detect(inputStream, metadata)` where metadata includes `TikaCoreProperties.RESOURCE_NAME_KEY` from filename
       - IMPORTANT: Use `MultipartFile.getBytes()` approach in the caller to avoid InputStream exhaustion (Pitfall 1). This service receives the stream but the caller must handle re-readability.

    2. **Content type validation:**
       - `fun validateContentType(domain: StorageDomain, contentType: String)` -- throws `ContentTypeNotAllowedException` if not in domain's allowlist

    3. **File size validation:**
       - `fun validateFileSize(domain: StorageDomain, fileSize: Long)` -- throws `FileSizeLimitExceededException` if over limit

    4. **SVG sanitization** using `io.github.borewit.sanitize.SVGSanitizer`:
       - `fun sanitizeSvg(input: ByteArray): ByteArray` -- strips script tags, event handlers, embedded JS
       - Returns sanitized SVG bytes

    5. **Storage key generation:**
       - `fun generateStorageKey(workspaceId: UUID, domain: StorageDomain, contentType: String): String`
       - Format: `{workspaceId}/{domain.name.lowercase()}/{UUID.randomUUID()}.{ext}`
       - Derive extension from MIME type using Tika's `MimeTypes.getDefaultMimeTypes().forName(contentType).extension` (falls back to empty string)
       - Strip leading dot from Tika extension if present

    Write tests FIRST (RED), then implement (GREEN). Use `@SpringBootTest(classes = [ContentValidationService::class])` pattern. For tests needing real byte content, create small test byte arrays (JPEG magic bytes: `0xFF 0xD8 0xFF`, PNG: `0x89 0x50 0x4E 0x47`).
  </action>
  <verify>
    <automated>./gradlew test --tests "riven.core.service.storage.ContentValidationServiceTest" 2>&1 | tail -20</automated>
  </verify>
  <done>ContentValidationService detects MIME types via magic bytes (not extension). Rejects disallowed types and oversized files with correct exceptions. SVGs are sanitized. Storage keys are UUID-based. All tests pass.</done>
</task>

  <behavior>
    - upload: Writes file to {basePath}/{key} and returns StorageResult with correct metadata
    - upload: Creates parent directories if they don't exist
    - download: Returns DownloadResult with file content, content type, and size
    - download: Throws StorageNotFoundException for non-existent key
    - delete: Removes file from disk (no error if already absent)
    - exists: Returns true for existing file, false for missing
    - healthCheck: Returns true when base directory is writable
    - healthCheck: Returns false when base directory is not writable
    - Path traversal: Rejects keys containing ".." segments
  </behavior>
  <action>
    Create `LocalStorageProvider` as a `@Service` with:
    - `@ConditionalOnProperty(name = ["storage.provider"], havingValue = "local")` (per CONTEXT.md decision)
    - Constructor injection: `KLogger`, `StorageConfigurationProperties`

    1. **upload:** Resolve full path from `basePath + key`. Verify resolved path starts with basePath (path traversal prevention per Pitfall 5). Create parent directories with `Files.createDirectories()` (safe per Pitfall 2). Write content via `Files.copy(content, path)`. Return `StorageResult`.

    2. **download:** Resolve and verify path. If not exists, throw `StorageNotFoundException`. Return `DownloadResult` with `Files.newInputStream(path)`, probed content type via `Files.probeContentType()`, and file size via `Files.size()`.

    3. **delete:** Resolve and verify path. Use `Files.deleteIfExists(path)` -- no error if already gone (idempotent).

    4. **exists:** Resolve path, return `Files.exists(path)`.

    5. **generateSignedUrl:** This method is a STUB in the local adapter -- it will be implemented by `SignedUrlService` and wired through `StorageService`. For now, throw `UnsupportedOperationException("Use SignedUrlService for local signed URLs")`. The `StorageService` (Plan 03) will handle URL generation by calling `SignedUrlService` directly rather than going through the provider interface for this operation.

    6. **healthCheck:** Check `Files.isWritable(basePath)`. Wrap in try-catch, return false on any exception.

    7. **Path validation private method:** `private fun resolveAndValidate(key: String): Path` -- resolves key against basePath, uses `normalize()`, checks `startsWith(basePath)`, throws `StorageProviderException` if traversal detected.

    Write tests FIRST (RED), then implement (GREEN). Use `@TempDir` JUnit annotation for isolated filesystem in tests. Do NOT use `@SpringBootTest` -- test as a plain unit with a manually constructed instance using a temp directory.
