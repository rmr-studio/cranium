## Decisions Made

1. **`hypersistence-utils-hibernate-71` (not `-70`).** The -70 artifact compiles against Hibernate 7.0.10; Boot 4.0.5 BOM ships 7.2.7, which made `AbstractClassJavaType.getJavaTypeClass` final. -71 targets 7.2.x.
2. **`hibernate-vector:7.2.7.Final` pinned to Boot 4.0.5 BOM's exact hibernate.version.** Avoids another minor-drift ClassChangeError surface.
3. **Resilience4j stays on 2.3.0 across all three direct coordinates** â€” `resilience4j-spring` has no 2.4.0 release; uniform 2.3.0 preserves binary compat.
4. **Temporal 1.34.0 accepted despite its POM listing Boot 2.7.18 parent.** In-project Boot 4 BOM overrides. Any runtime breakage is Plan 03/04 surface per CONTEXT.md's policy of narrow per-plan scope.
5. **Security 7 `SecurityFilterChain` idioms NOT migrated here â€” strictly Plan 02's scope.** Only fixed compile-blocking nullability drift.
6. **Removed explicit `hibernate.dialect` config** â€” Hibernate 7 HHH90000025 flags this; driver-based auto-detection is sufficient.
