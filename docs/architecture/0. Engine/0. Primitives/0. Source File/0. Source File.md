A **source file** is the engine's record of one externally-identifiable artifact (or one self-contained synthesis) that pages will cite. Most of the type, it is not the artifact itself (ie. Slack thread, Github code file) it is a structured pointer + provenance envelope around it. However can sometimes be the artifact during niche cases, such as manual direct ingestion of engineer tribal knowledge, or ingestion of information that has no tangible provenance (ie. Thoughts, agreed decisions from meetings, reflections). Every source file holds the backbone that lets atomic pages (L3), community summaries (L2), routing (L1), and the drift loop (L5) operate without re-reading the underlying world on every query.

The primitive must answer five operational questions deterministically:

1. **Where does the canonical text live?** (URI + system-of-record)
2. **Who has write authority over it?** (governs how drift is detected)
3. **What slots inside it do pages cite?** (regions + hashes)
4. **How much should the engine trust it?** (confidence weight)
5. **What invalidates pages derived from it?** (drift signal + signal source)

Five **source classes** exist; the class determines how each universal characteristic specializes. See [[source-class-taxonomy]] (Meta) for the framework.

---

## Universal Characteristics

Every source file, regardless of class, must store these fields. The engine reads them in the order listed; later fields depend on earlier ones being well-formed.

### `Identifier`

- **`source_id`** ‚Äî UUID v5 derived from `(workspace_id, canonical_uri)`. Stable across re-ingests. Deterministic so two ingest paths cannot create duplicates.
- **`canonical_uri`** ‚Äî the smallest URI a teammate would link to in chat when answering a question, or if the application needs to reference exact data to back up claims and answer queries (ie. Pulling an exact conversation from slack to support a claim). 
	- For classes without an external URI (native, digest, reflection), this is a Cranium-internal URN (`cranium://workspace/<id>/native/<slug>`).
- **`display_name`** ‚Äî human-readable label for surfaces (search results, citation footers).

### `Source Class`

The discriminator. Drives every downstream behavior. One of:

- `live_pointer` ‚Äî external SoR, hash-tracked
- `native` ‚Äî Cranium-internal SoR, edit-event-tracked
- `hybrid` ‚Äî external SoR + Cranium annotations, dual-tracked
- `digest` ‚Äî one-shot synthesis, no drift
- `reflection` ‚Äî personal note, no drift

Cannot be inferred at query time. Must be set at capture time and is immutable for the lifetime of the source file. Reclassification = new source file + migration of citing pages.

### `System of Record`
This is where the information  is derived from, and also dictates the write permissions that a source page has access to (ie. Which source pages can be edited, compared to which source pages must not be touched unless the original source of truth is also recognised to be altered or deleted)
This controls the logic of drift behaviour and checking, determining which source pages are continuously monitored against their original source of truth to identify drift or stale data
A source page can have **multiple** systems of records
- **`sor_kind`** ‚Äî enum: `external` | `cranium` | `external+cranium` | `none`.
- **`sor_pointer`** ‚Äî for `external` and `external+cranium`, the upstream reference (Slack channel + thread ts, GitHub repo + path + ref, Notion page id, etc.). Null for `native`, `digest`, `reflection`.
- **`write_authority`** ‚Äî who can change the canonical text. This is the invariant ([[source-class-taxonomy]]): write-authority determines class, not where bits are cached.

### `Connector Lane`
This represents the original pipeline or job that was used to originally ingest (ie. Where it was pulled and where the real artifacts are currently being stored).

- **`lane`** ‚Äî which ingest pipeline this source entered through. Examples: `github`, `slack`, `notion`, `linear`, `posthog`, `dovetail`, `cranium-native`, `cranium-digest`.
- **`lane_metadata`** ‚Äî lane-specific blob (Slack channel id, GitHub repo + branch, Notion workspace, etc.) needed to refresh/resync.
- **`ingest_actor`** ‚Äî connector identity (`nango:slack:<connection_id>`) or user id for native/digest/reflection.

The lane is what the L1 routing manifest filters on for retrieval scoping ("answer this from `lane: github` sources only").

### `Confidence Weight`
The confidence weight is an LLM reasoned approximation of the signal quality that the source provides, and how much this information biases the outcome of queries, indexing and aggregations to form trends, patterns and analysis

Per [[source-confidence-weighting]]. Numeric float in `[0.0, 1.0]`. Per-class defaults:

| Class | Default | Notes |
|---|---|---|
| live_pointer (code) | 1.0 | Compiler-enforced |
| live_pointer (ADR/design doc) | 0.8 | Authored, reviewed |
| hybrid | 0.7 | Upstream + curation |
| native | 0.6 | SME-asserted, no compiler |
| live_pointer (Confluence/Notion) | 0.4 | Untracked drift |
| digest | 0.3 | One-shot synthesis, no drift signal |
| live_pointer (Slack, unflagged) | 0.05 | Mostly noise |
| reflection | personal | Never surfaces in shared retrieval |

Weight feeds the retrieval ranker and gates auto-promotion to canonical pages.

### `Regions`

The list of hashable content slots inside the source. **One source ‚Üí many regions.** Each region is the smallest text slot a page would cite.
Only source pages that reference external sources will have regions (ie. `live_pointer` and `hybrid`) As they reference a source of truth outside of the system, so the hash will need to be used to detect drift

Region shape:

```yaml
regions:
  - region_id: <stable slug or generated id>
    locator:                          # how to find the slot inside the source
      kind: line_range | symbol | heading | ts | block_id | xpath
      value: <kind-specific>
    region_class: upstream | native   # for hybrid; else same as source class
    content_hash: <sha256>            # of the cited text
    last_hashed_at: <iso8601>
    cited_by: [<page_id>, ...]        # back-pointer maintained by ingest
```

Region rules:
- A region is **content-addressed by hash**, not by locator. Locator can shift (line-number drift); hash detects whether the *text* moved.
- A page cites `(source_id, region_id)`, never a raw locator. Locator changes don't break citations.
- Hash is over normalized text (LF line endings, no trailing whitespace) so cosmetic edits don't false-positive.

### `Hash Skip`

- **`hash_skip_eligible`** ‚Äî boolean. True only for `live_pointer` and the upstream regions of `hybrid`. False for `native`, `digest`, `reflection`, and hybrid native-regions.
- **Skip ledger key** ‚Äî `(workspace_id, source_id, region_id) ‚Üí content_hash`. On resync, recomputed hash equals ledger hash ‚Üí page skipped, zero LLM calls. See [[content-hash-skip-resync]].
- **Skip-rate floor invariant** ‚Äî workspace-level skip-rate on resync should clear ‚â•70%. Lower means the hash normalization broke (line endings, whitespace) and cost is silently inflating.

### `Drift Signal`

- **`drift_signal_kind`** ‚Äî enum: `rehash_diff` | `edit_event` | `dual` | `none`.
- **`drift_subscriber`** ‚Äî for `edit_event` and `dual`, the event topic the L5 loop subscribes to (`cranium.native.edited`, `cranium.hybrid.annotation.edited`).
- **`drift_handler`** ‚Äî for `rehash_diff`, the scheduled resync job; for `edit_event`, the consumer; for `dual`, both. See [[native-source-drift]] for native; [[hybrid-source-dual-hash]] for hybrid.

### `Ingestion Metadata`

- **`first_ingested_at`** ‚Äî fixed at creation.
- **`last_synced_at`** ‚Äî most recent successful resync attempt.
- **`last_drift_event_at`** ‚Äî last time a region hash changed (or edit-event fired for native).
- **`sync_status`** ‚Äî enum: `ok` | `stale` | `broken_citation` | `class_mismatch`.

### `Pages Derived`

- **`pages_derived: [<page_id>, ...]`** ‚Äî back-references from this source to every L3 page that cites at least one of its regions. Maintained by the ingest writer, never by hand. Used by L5 to enqueue resynth on drift. This ensures that every time a source changes, every page that references this source is also updated to reflect the new information received

### `Lifecycle State`

- **`active`** ‚Äî boolean. Frozen sources (`active: false`) skip L0 + L5 entirely and surface only with a "frozen" badge in retrieval.
- **`frozen_reason`** ‚Äî when applicable (`retired_system`, `manual_snapshot`, `paper_published`, etc.).

## Per-Class Specialization

Each class fills the universal characteristics with class-specific values and adds a small set of class-only fields.

### Live Pointer

External SoR. The class L0 was originally designed for.

| Field | Live-pointer specifics |
|---|---|
| `write_authority` | external system (engineers via PR, Slack participants, Notion editors) |
| `canonical_uri` | external permalink (`https://github.com/<org>/<repo>/blob/<ref>/<path>`, Slack permalink, Notion page url) |
| `regions[*].region_class` | always `upstream` |
| Hash role | **drives L0 skip-on-resync.** Region hash unchanged ‚Üí page skipped, zero LLM calls. |
| Drift signal | upstream rehash diff (scheduled resync via the lane's connector) |
| Confidence default | varies by sub-type (see table above) |
| Resync trigger | scheduled per-lane (high-churn lanes more often), or webhook-driven when the connector supports it |

Class-only fields:

```yaml
upstream_ref:
  ref_kind: git_sha | slack_ts | notion_version_id | linear_revision
  ref_value: <sha or ts or id>
  fetched_at: <iso8601>
```

Sub-types and region locator conventions:

| Sub-type | Region locator kind | Region granularity |
|---|---|---|
| GitHub repo file | `symbol` (tree-sitter parse) | function body, class, top-level block |
| GitHub PR | `block_id` (description, diff hunk index) | description + each hunk |
| GitHub review | `block_id` (comment id) | per comment in thread |
| ADR / design doc | `heading` | section between headings, or `<whole-file>` for short docs |
| Slack thread | `ts` (message timestamps) | root + each reply |
| Notion page | `block_id` | block-level (Notion already has stable block ids) |
| Linear ticket | `block_id` (`description`, `state_transitions`) | description + state history |
| Linear comment | `block_id` (comment id) | per comment |
| PostHog event spec | `symbol` (event name) | per event definition |
| Dovetail insight | `block_id` (insight id) | per insight |

### Native

Cranium *is* the system of record. Created by `Write` capture verb.

| Field | Native specifics |
|---|---|
| `write_authority` | a Cranium user (with role permissions) |
| `canonical_uri` | `cranium://workspace/<id>/native/<slug>` |
| `regions[*].region_class` | always `native` |
| Hash role | **versioning / change detection**, never skip. Native sources never enter the L0 skip ledger. See [[native-source-drift]]. |
| Drift signal | Cranium DB edit event (`{source_id, region_id, edited_at, editor}`); plus weekly staleness scan (`Updated >6mo AND retrieved ‚â•N`) |
| Confidence default | 0.6 (SME-asserted) |
| Resync trigger | edit-event subscription, not scheduled |

Class-only fields:

```yaml
authored_by: <user_id>
last_edited_by: <user_id>
revision_count: <int>
sme_review_flag: open | none
sme_review_due_at: <iso8601 | null>
```

Native sub-types (informational, share the same primitive):

- **SME Q&A capture** ‚Äî root region = question, each answer/follow-up region.
- **Glossary entry** ‚Äî single region or short heading list.
- **"Why we don't do X" note** ‚Äî single region.
- **Decision rationale typed in directly** ‚Äî root region + alternatives considered.
- **Onboarding tribal knowledge** ‚Äî heading-keyed regions.

### Hybrid

External SoR + Cranium-stored annotations. Created by `Capture` verb.

| Field | Hybrid specifics |
|---|---|
| `write_authority` | dual: external for upstream regions, Cranium user for annotation regions |
| `canonical_uri` | external permalink (the upstream is the anchor identity) |
| `regions[*].region_class` | mixed ‚Äî some `upstream`, some `native` |
| Hash role | **dual.** Upstream regions follow live-pointer L0; native regions follow native-source drift. See [[hybrid-source-dual-hash]]. |
| Drift signal | upstream rehash diff **AND** Cranium edit events on annotation regions, joined at L5 |
| Confidence default | 0.7 (upstream provenance + SME curation) |
| Resync trigger | both (whichever side changes) |

Class-only fields:

```yaml
upstream_ref: { ... }                    # same shape as live_pointer
annotation_count: <int>
annotation_authors: [<user_id>, ...]
```

Region table example (Slack thread imported + annotated):

```yaml
regions:
  - region_id: root
    locator: { kind: ts, value: "1715000000.001" }
    region_class: upstream
    content_hash: <sha>
  - region_id: reply-3
    locator: { kind: ts, value: "1715000300.004" }
    region_class: upstream
    content_hash: <sha>
  - region_id: annotation-1
    locator: { kind: block_id, value: "annot-1" }
    region_class: native
    content_hash: <sha>
    authored_by: <user_id>
```

### Digest

Self-contained synthesis aggregating multiple upstream sources. No live URI. Created via `Write ‚Üí kind: digest`.

| Field | Digest specifics |
|---|---|
| `write_authority` | author (user_id) |
| `canonical_uri` | `cranium://workspace/<id>/digest/<slug>` |
| `regions[*].region_class` | `native` (body is hand-authored) |
| Hash role | **none for skip math.** The L0 engine ignores digests during resync cost accounting. |
| Drift signal | none automatic; manual revisit only |
| Confidence default | 0.3 |
| Parents required? | **Yes if declarable.** Digests must declare any live-pointer parents so retrieval can flag staleness. |

Class-only fields:

```yaml
parents:                                  # upstream sources this digest synthesized from
  - source_id: <id> | <unconnectable_marker>
    parent_kind: live_pointer | external_unconnectable | synthesized
synthesized_at: <iso8601>
re_synthesis_recommended_after: <iso8601 | null>   # author-declared TTL
```

If `parents` is empty *and* `parent_kind: synthesized` for all, the digest is a pure abstraction (e.g. a literature review with no Cranium-side primary sources). Allowed, but tagged `unverifiable` in retrieval.

### Reflection

Personal note. Like native, but scoped to one user and never surfaces in shared retrieval without an explicit promotion step.

| Field | Reflection specifics |
|---|---|
| `write_authority` | single user_id; no shared edit |
| `canonical_uri` | `cranium://user/<id>/reflection/<slug>` |
| `regions[*].region_class` | `native` |
| Hash role | none |
| Drift signal | none |
| Confidence default | `personal` (sentinel, not a number) ‚Äî never enters shared ranker |
| Promotion path | reflection ‚Üí native (via explicit "share" action that re-classes the source file) |

Class-only fields:

```yaml
owner: <user_id>
visibility: private | shared-by-promotion
promoted_to: <source_id | null>           # if reflection was elevated to native
```

## Invariants

The primitive enforces these at write-time. Violations are programmer errors, not data errors.

1. **Class immutable.** Source class cannot be edited after creation. Reclassification = new source + migration of `cited_by` references.
2. **Write-authority determines class.** If `write_authority` includes a Cranium user but `sor_kind = external`, the source is `hybrid`, not `live_pointer`. The engine should refuse to save the inconsistent combination.
3. **Region hash covers cited text only.** Hash over the slot the citing page actually reads. Not the whole file (false positives), not a paragraph downstream (false negatives).
4. **`pages_derived` is the source-of-truth for back-references.** L5 reads it on every drift signal to enqueue resynth. Manual edits forbidden; ingest writer maintains it.
5. **Native + digest + reflection never enter the L0 skip ledger.** Putting them there breaks both the skip math (false skip-rate) and the drift loop (no signal to act on).
6. **Hybrid must split `regions[].region_class`.** A flat region list with all `upstream` or all `native` is a misclassified live-pointer or native, not a hybrid.
7. **Active = false ‚áí no L0, no L5.** Frozen sources are read-only artifacts; the engine treats them as terminal and skips them everywhere except retrieval display.

## Anti-Patterns

Recurring mislabel patterns and the fix:

| Symptom | Root cause | Fix |
|---|---|---|
| Hand-written essay synthesizing several papers, lives in `sources/`, body contains the prose | Digest mislabeled as live_pointer | Re-class as `digest`; add `parents` (even if `parent_kind: external_unconnectable`) |
| SME types a Q&A capture, engine tries to hash it against an external Slack URL | Native mislabeled as live_pointer | Re-class as `native`; route through edit-event drift |
| Slack thread imported, SME adds notes, single flat region list with one hash | Hybrid flattened to live_pointer | Split regions into `upstream` + `native` sets; dual-hash per [[hybrid-source-dual-hash]] |
| Confluence page treated with `confidence: 1.0` because "it's documentation" | Source-class blind weighting | Reapply [[source-confidence-weighting]]; Confluence ~0.4 |
| Personal note showing up in shared retrieval | Reflection set to `shared` without explicit promotion | Enforce reflection ‚Üí native promotion as a state transition, not a flag flip |
