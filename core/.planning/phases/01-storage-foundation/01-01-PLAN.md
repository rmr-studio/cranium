       - `workspaceId: UUID` (column `workspace_id`, not null, uuid)
       - `domain: StorageDomain` with `@Enumerated(EnumType.STRING)`
       - `storageKey: String` (column `storage_key`, not null, unique)
       - `originalFilename: String` (column `original_filename`, not null)
       - `contentType: String` (column `content_type`, not null)
       - `fileSize: Long` (column `file_size`, not null)
       - `uploadedBy: UUID` (column `uploaded_by`, not null, uuid)
       - `@Table(name = "file_metadata")` with indexes on `workspace_id`, composite `workspace_id, domain`, and unique on `storage_key`
       - Implement `toModel(): FileMetadata` mapping all fields including inherited `createdAt`/`updatedAt`

    2. Create `FileMetadataRepository` interface in `repository/storage/` extending `JpaRepository<FileMetadataEntity, UUID>`:
       - `fun findByWorkspaceIdAndDomain(workspaceId: UUID, domain: StorageDomain): List<FileMetadataEntity>`
       - `fun findByWorkspaceId(workspaceId: UUID): List<FileMetadataEntity>`
       - `fun findByStorageKey(storageKey: String): Optional<FileMetadataEntity>`
       - `fun findByIdAndWorkspaceId(id: UUID, workspaceId: UUID): Optional<FileMetadataEntity>`

    3. Create `db/schema/01_tables/storage.sql` with `file_metadata` table DDL:
       - Follow existing SQL file conventions in `db/schema/01_tables/`
       - Columns: `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`, `workspace_id UUID NOT NULL`, `domain VARCHAR(50) NOT NULL`, `storage_key VARCHAR(500) NOT NULL UNIQUE`, `original_filename VARCHAR(500) NOT NULL`, `content_type VARCHAR(255) NOT NULL`, `file_size BIGINT NOT NULL`, `uploaded_by UUID NOT NULL`, `deleted BOOLEAN NOT NULL DEFAULT FALSE`, `deleted_at TIMESTAMPTZ`, `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`, `updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`

    4. Create `db/schema/02_indexes/storage_indexes.sql`:
       - `idx_file_metadata_workspace_id` on `workspace_id`
       - `idx_file_metadata_workspace_domain` on `workspace_id, domain`
       - Unique index `uq_file_metadata_storage_key` on `storage_key` (belt-and-suspenders with column constraint)

    5. Add storage exceptions to `ArgumentExceptions.kt`:
       - `open class StorageException(message: String, cause: Throwable? = null) : RuntimeException(message, cause)` (base class)
       - `class StorageNotFoundException(message: String) : StorageException(message)` (file not found in provider)
       - `class StorageProviderException(message: String, cause: Throwable? = null) : StorageException(message, cause)` (provider I/O error)
       - `class ContentTypeNotAllowedException(message: String) : StorageException(message)` (MIME type rejected)
       - `class FileSizeLimitExceededException(message: String) : StorageException(message)` (file too large)
       - `class SignedUrlExpiredException(message: String) : StorageException(message)` (token expired or invalid)

    6. Add exception handlers to `ExceptionHandler.kt`:
       - `ContentTypeNotAllowedException` -> 415 UNSUPPORTED_MEDIA_TYPE
       - `FileSizeLimitExceededException` -> 413 PAYLOAD_TOO_LARGE
       - `StorageNotFoundException` -> 404 NOT_FOUND
       - `SignedUrlExpiredException` -> 403 FORBIDDEN
       - `StorageProviderException` -> 502 BAD_GATEWAY (provider failure)
       - Follow existing handler pattern (ErrorResponse, storeExceptionForAnalytics, logger)
