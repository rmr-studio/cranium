   - `data class S3(val bucket: String = "riven-storage", val region: String = "us-east-1", val accessKeyId: String = "", val secretAccessKey: String = "", val endpointUrl: String? = null)` -- all S3 config. endpointUrl is nullable (null = standard AWS S3).

2. **Create SupabaseStorageProvider** at `src/main/kotlin/riven/core/service/storage/SupabaseStorageProvider.kt`:
   - `@Service` + `@ConditionalOnProperty(name = ["storage.provider"], havingValue = "supabase")`
   - Constructor injection: `KLogger`, `SupabaseClient`, `StorageConfigurationProperties`
   - Access bucket via `supabaseClient.storage.from(storageConfig.supabase.bucket)` -- store as lazy val or compute per call.
   - Follow LocalStorageProvider section organization: `// ------ Public Operations ------` and `// ------ Private Helpers ------`.
   - Every method wraps supabase-kt suspend calls in `runBlocking {}` per user decision.
   - **upload**: Read `content.readAllBytes()`, call `bucket.upload(key, bytes) { upsert = true; contentType(contentType) }` inside runBlocking. Return `StorageResult(storageKey=key, contentType=contentType, contentLength=contentLength)`. Catch exceptions -> `StorageProviderException`.
   - **download**: Call `bucket.downloadAuthenticated(key)` inside runBlocking. Returns ByteArray. Wrap in `ByteArrayInputStream`. ContentType from supabase-kt download is not available, so use `"application/octet-stream"` (per research pitfall 1 -- StorageService looks up actual content type from metadata when needed). Return `DownloadResult`. Catch not-found -> `StorageNotFoundException`, others -> `StorageProviderException`.
   - **delete**: Call `bucket.delete(key)` inside runBlocking. Idempotent -- do not throw on missing file. Catch exceptions -> `StorageProviderException`.
   - **exists**: Try `bucket.list(key)` or attempt download head check. Simplest: use a try/catch on `downloadAuthenticated` or list with prefix matching. Use whatever supabase-kt provides. If unclear, list with prefix and check if result contains the key.
   - **generateSignedUrl**: Call `bucket.createSignedUrl(path = key, expiresIn = expiresIn)` inside runBlocking. Return the URL string. Catch exceptions -> `StorageProviderException`.
   - **healthCheck**: Try `supabaseClient.storage.retrieveBucketById(bucketName)`. If throws (bucket doesn't exist), auto-create with `supabaseClient.storage.createBucket(id = bucketName) { public = false }`. Return true on success, false on any exception (log warning). Per user decision.
   - For not-found detection: create a `private fun isNotFound(e: Exception): Boolean` helper that checks for relevant supabase-kt exception types/messages (e.g. contains "not found" or "404" or is a specific supabase-kt exception type). Research indicates checking exception message content.
   - Exception messages must be generic (no provider details) per user decision. Provider exception goes in `cause` only.

3. **Update application.yml** -- add default config for `storage.supabase.bucket` and `storage.s3.*` under the existing `storage:` section.

4. **Update application-test.yml** -- ensure test profile has `storage.provider: local` explicitly set (should already be the case) and add default `storage.supabase.bucket` and `storage.s3.*` so tests don't fail on missing config even when loading the full config class.
