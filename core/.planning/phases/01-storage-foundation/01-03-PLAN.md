    - validateToken: Valid token returns storageKey and expiresAt
    - validateToken: Expired token returns null
    - validateToken: Tampered token (modified signature) returns null
    - validateToken: Tampered token (modified storageKey) returns null
    - validateToken: Malformed token (not Base64, wrong format) returns null

  <action>
    Create `SignedUrlService` as a `@Service` with constructor-injected `KLogger` and `StorageConfigurationProperties`:

    1. **Token generation** (`fun generateToken(storageKey: String, expiresIn: Duration): String`):
       - Clamp `expiresIn` to `maxExpirySeconds` if it exceeds the bound
       - Compute expiry timestamp: `Instant.now().plus(expiresIn).epochSecond`
       - Payload: `"$storageKey:$expiresAt"`
       - HMAC-SHA256 the payload using `storage.signed-url.secret`
       - Encode as Base64URL: `Base64.getUrlEncoder().withoutPadding().encodeToString("$payload:$signature".toByteArray())`

    2. **Token validation** (`fun validateToken(token: String): Pair<String, Long>?`):
       - Decode Base64URL
       - Split on last colon to get payload vs signature
       - Recompute HMAC on payload
       - **CRITICAL:** Use `MessageDigest.isEqual()` for constant-time comparison (Pitfall 3)
       - Parse storageKey and expiresAt from payload
       - Check expiry: `Instant.now().epochSecond <= expiresAt`
       - Return `Pair(storageKey, expiresAt)` or null

    3. **URL generation** (`fun generateDownloadUrl(storageKey: String, expiresIn: Duration): String`):
       - Calls `generateToken()` and returns `"/api/v1/storage/download/$token"`

    4. **Default expiry helper** (`fun getDefaultExpiry(): Duration`):
       - Returns `Duration.ofSeconds(config.signedUrl.defaultExpirySeconds)`

    Write tests FIRST (RED), then implement (GREEN). Tests should NOT use `@SpringBootTest` -- construct the service directly with a test config. Use a fixed secret for deterministic HMAC output in some tests, and time-based assertions for expiry tests.

    **Refactor** existing `StorageService` stub (currently just injects `SupabaseClient`) into a full orchestrating service.

    Constructor dependencies:
    - `private val logger: KLogger`

    - `private val contentValidationService: ContentValidationService`

    - `private val authTokenService: AuthTokenService`

    **Public methods (all with `@PreAuthorize("@workspaceSecurity.hasWorkspace(#workspaceId)")`)**:

    1. `fun uploadFile(workspaceId: UUID, domain: StorageDomain, file: MultipartFile): UploadFileResponse`
       - `val userId = authTokenService.getUserId()`
       - Read file bytes: `val bytes = file.bytes` (solves InputStream exhaustion -- Pitfall 1)
       - Detect content type: `contentValidationService.detectContentType(ByteArrayInputStream(bytes), file.originalFilename)`
       - Validate content type and file size against domain
       - If SVG: sanitize bytes, use sanitized bytes for storage
       - Generate storage key via contentValidationService
       - Upload to provider: `storageProvider.upload(key, ByteArrayInputStream(content), detectedType, content.size.toLong())`
       - Persist `FileMetadataEntity` via repository
       - Log `Activity.FILE_UPLOAD` activity
       - Generate signed URL via signedUrlService
       - Return `UploadFileResponse(metadata.toModel(), signedUrl)`

    2. `fun getFile(workspaceId: UUID, fileId: UUID): FileMetadata`
       - `ServiceUtil.findOrThrow { fileMetadataRepository.findByIdAndWorkspaceId(fileId, workspaceId) }.toModel()`

    3. `fun generateSignedUrl(workspaceId: UUID, fileId: UUID, expiresInSeconds: Long?): SignedUrlResponse`
       - Find metadata, generate URL with custom or default expiry
       - Return `SignedUrlResponse(url, expiresAt)`

    4. `fun deleteFile(workspaceId: UUID, fileId: UUID)`
       - `val userId = authTokenService.getUserId()`
       - Find metadata entity, soft-delete it (`deleted = true, deletedAt = ZonedDateTime.now()`)
       - Save entity
       - Delete physical file: `storageProvider.delete(entity.storageKey)` -- if this fails, log error but don't roll back soft-delete (Pitfall 4)
       - Log `Activity.FILE_DELETE` activity

    5. `fun listFiles(workspaceId: UUID, domain: StorageDomain?): FileListResponse`
       - If domain provided: `fileMetadataRepository.findByWorkspaceIdAndDomain(workspaceId, domain)`
       - Else: `fileMetadataRepository.findByWorkspaceId(workspaceId)`
       - Map to models, return `FileListResponse`

    6. `fun downloadFile(token: String): DownloadResult`
       - Validate token via `signedUrlService.validateToken(token)` -- throw `SignedUrlExpiredException` if null
       - Fetch file from provider: `storageProvider.download(storageKey)`
       - Look up metadata for original filename: `fileMetadataRepository.findByStorageKey(storageKey)`
       - Return `DownloadResult` with original filename from metadata
