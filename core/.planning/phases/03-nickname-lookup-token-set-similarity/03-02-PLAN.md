**1. Add NAME branch in findCandidates() loop (after the PHONE branch, around line 77):**

For NAME signals, do two things:
a) Run nickname candidate query and add results to allCandidates
b) Re-score trigram candidates using token overlap

```kotlin
// After the existing PHONE branch:
if (signalType == MatchSignalType.NAME) {
    // Token-set re-scoring on trigram candidates
    val rescored = candidates.map { candidate ->
        val tokenOverlap = TokenSimilarity.overlap(normalizedValue, candidate.candidateValue)
        val finalScore = maxOf(candidate.similarityScore, tokenOverlap)
        if (finalScore > candidate.similarityScore) {
            candidate.copy(similarityScore = finalScore)
        } else {
            candidate
        }
    }
    // Replace trigram candidates with rescored versions
    allCandidates.removeAll(candidates.toSet())
    allCandidates.addAll(rescored)

    // Nickname expansion query
    val nicknameCandidates = findNicknameCandidates(triggerEntityId, workspaceId, normalizedValue, signalType)
    allCandidates.addAll(nicknameCandidates)
}
```

IMPORTANT: The trigram `candidates` are already added to `allCandidates` on line 71. For NAME signals, you need to either: (a) restructure so trigram candidates are added AFTER re-scoring, or (b) remove and re-add. The cleaner approach is to restructure the loop so that for NAME signals, trigram results are rescored BEFORE being added:

```kotlin
val candidates = runCandidateQuery(...)

// Re-score NAME trigram candidates with token overlap BEFORE adding
val processedCandidates = if (signalType == MatchSignalType.NAME) {
    candidates.map { candidate ->
        val tokenOverlap = TokenSimilarity.overlap(normalizedValue, candidate.candidateValue)
        val finalScore = maxOf(candidate.similarityScore, tokenOverlap)
        if (finalScore > candidate.similarityScore) candidate.copy(similarityScore = finalScore) else candidate
    }
} else {
    candidates
}
allCandidates.addAll(processedCandidates)

// Signal-type-specific additional queries
if (signalType == MatchSignalType.PHONE) { ... }
if (signalType == MatchSignalType.NAME) {
    val nicknameCandidates = findNicknameCandidates(triggerEntityId, workspaceId, normalizedValue, signalType)
    allCandidates.addAll(nicknameCandidates)
}
```

**2. Add findNicknameCandidates() private method:**

Follow the exact pattern of findPhoneExactDigitsCandidates(). Parameters: triggerEntityId, workspaceId, normalizedValue, signalType.

- Tokenize normalizedValue by whitespace, lowercase, filter empty
- For each token, call NicknameExpander.expand(token) and collect all variants + original tokens into a flat Set
- If variants set is empty (no known nicknames for any token), return emptyList() immediately — do NOT run SQL with empty IN-clause
- Run native SQL query with IN-clause matching `LOWER(ea.value->>'value') IN (:variants)` — see research Pattern 3 for exact SQL
- Fixed similarity score of 0.95
- matchSource = MatchSource.NICKNAME
- Add import for NicknameExpander and TokenSimilarity (same package, but explicit import)

**3. Update mergeCandidates() tie-breaking:**

Change from `maxByOrNull { it.similarityScore }` to `maxWithOrNull(compareBy<CandidateMatch> { it.similarityScore }.thenBy { if (it.matchSource == MatchSource.NICKNAME) 1 else 0 })`. This prefers NICKNAME over TRIGRAM when similarity scores are equal.
