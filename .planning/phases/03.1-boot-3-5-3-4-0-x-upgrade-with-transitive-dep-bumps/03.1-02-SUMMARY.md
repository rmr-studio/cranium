## Accomplishments

- Migrated `SecurityConfig.securityFilterChain` from Security 6.x fluent-lambda form to Security 7 Kotlin DSL (`http { ... }` via `org.springframework.security.config.annotation.web.invoke`)
- Replaced `.requestMatchers(...).permitAll()` / `.anyRequest().authenticated()` chain with `authorize(pattern, permitAll)` / `authorize(anyRequest, authenticated)` statements
- Converted `oauth2ResourceServer { oauth2 -> oauth2.jwt { jwt -> jwt.jwtAuthenticationConverter(tokenDecoder) } }` to property-style `oauth2ResourceServer { jwt { jwtAuthenticationConverter = tokenDecoder } }`
- Migrated exception handlers to property-assignment via SAM conversion (`authenticationEntryPoint = AuthenticationEntryPoint { ... }`, `accessDeniedHandler = AccessDeniedHandler { ... }`)
- `./gradlew compileKotlin` clean (no Security deprecation warnings on migrated surface); auth/security/JWT test slice BUILD SUCCESSFUL

## Before / After (Key Lines)

**Before (Security 6 fluent-lambda form):**
```kotlin
http
    .cors { it.configurationSource(corsConfig()) }
    .csrf { it.disable() }
    .sessionManagement { it.sessionCreationPolicy(SessionCreationPolicy.STATELESS) }
    .authorizeHttpRequests { auth ->
        auth
            .requestMatchers("/api/auth/**").permitAll()
            // ...
            .anyRequest().authenticated()
    }
    .oauth2ResourceServer { oauth2 ->
        oauth2.jwt { jwt -> jwt.jwtAuthenticationConverter(tokenDecoder) }
    }
    .exceptionHandling { exceptions ->
        exceptions
            .authenticationEntryPoint { _, response, authException -> ... }
            .accessDeniedHandler { _, response, accessDeniedException -> ... }
    }
```

**After (Security 7 Kotlin DSL `http { }` invoke form):**
```kotlin
http {
    cors { configurationSource = corsConfig() }
    csrf { disable() }
    sessionManagement { sessionCreationPolicy = SessionCreationPolicy.STATELESS }
    authorizeHttpRequests {
        authorize("/api/auth/**", permitAll)
        // ...
        authorize(anyRequest, authenticated)
    }
    oauth2ResourceServer {
        jwt { jwtAuthenticationConverter = tokenDecoder }
    }
    exceptionHandling {
        authenticationEntryPoint = AuthenticationEntryPoint { _, response, authException ->
            response.sendError(HttpServletResponse.SC_UNAUTHORIZED, authException.message)
        }
        accessDeniedHandler = AccessDeniedHandler { _, response, accessDeniedException ->
            response.sendError(HttpServletResponse.SC_FORBIDDEN, accessDeniedException.message)
        }
    }
}
```

## Creative Adaptations

- **Exception handlers via SAM conversion.** The Security 7 Kotlin DSL exposes `authenticationEntryPoint` and `accessDeniedHandler` as assignable properties rather than builder methods. Assigned `AuthenticationEntryPoint { ... }` / `AccessDeniedHandler { ... }` SAM lambdas directly — the explicit SAM interface reference is required because Kotlin cannot resolve the functional type from property position alone. Explicit imports added (`org.springframework.security.web.AuthenticationEntryPoint`, `org.springframework.security.web.access.AccessDeniedHandler`).
- **`jwtAuthenticationConverter` as property, not method.** The Kotlin DSL's `JwtDsl` exposes `jwtAuthenticationConverter` as a `var`, so `jwt { jwtAuthenticationConverter = tokenDecoder }` replaces `jwt.jwtAuthenticationConverter(tokenDecoder)`. Semantically identical; DSL-idiomatic.
- **CORS source assignment.** `cors { configurationSource = corsConfig() }` replaces `.cors { it.configurationSource(corsConfig()) }`. Same bean, property-style binding.

- **Imports tightened.** Dropped unused `org.springframework.boot.context.properties.ConfigurationProperties`; flattened fully-qualified `org.springframework.web.cors.*` usages to top-of-file imports for readability. Cosmetic, not behavioural.
