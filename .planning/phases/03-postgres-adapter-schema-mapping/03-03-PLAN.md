      - Steps:
        1. Fetch + decrypt connection credentials (ServiceUtil.findOrThrow).
        2. Call `postgresAdapter.introspectWithFkMetadata(PostgresCallContext(workspaceId, connectionId))`.
        3. Load stored table mappings + field mappings for this connectionId (single batched query per repo).
        4. For each live table: compute fresh hash; compare to stored hash → driftStatus. For each live column: resolve existing mapping row, FK metadata, auto-detect cursor heuristic (name ILIKE `updated_at`/`modified_at`/`last_modified` AND pgType ILIKE `timestamp%`), and suggestedSchemaType via PgTypeMapper.toSchemaType. For columns in stored but not live: mark stale=true via repository.save (transactional side-effect).
        5. For the chosen cursor column (or the auto-detected default if none chosen), call CursorIndexProbe.isIndexed. If false, populate cursorIndexWarning on the table response.
        6. Persist updated schemaHash + lastIntrospectedAt on each stored table mapping (but NOT published — that only flips on explicit Save).
        7. Return the full response.
      - Private helpers: `autoDetectCursorColumn(columns): ColumnSchema?`, `resolveDriftStatus(stored, fresh): DriftStatus`, `markDroppedColumnsStale(stored, fresh)`.
