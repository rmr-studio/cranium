Create two Kotlin `object` singletons in `riven.core.service.identity` package. These are NOT Spring beans — pure stateless utilities.

**NicknameExpander.kt:**
- Kotlin `object NicknameExpander` (per user decision — not a Spring bean)
- Private `GROUPS: List<Set<String>>` companion-style constant with ~150 English name groups. Each set contains all known variants (e.g. `setOf("william", "bill", "will", "willy", "liam")`). All names stored lowercase.
- Private `INDEX: Map<String, Set<String>>` — reverse lookup map built at object init via `buildIndex()`. Every name variant maps to its full group set.
- `fun expand(name: String): Set<String>` — looks up `name.lowercase()` in INDEX, returns `group - name.lowercase()`. Unknown names return `emptySet()`.
- `fun areEquivalent(a: String, b: String): Boolean` — checks if both names are in the same group via INDEX lookup.
- Include ~150 name groups covering common English nicknames. Use the research file's examples as a starting point and extend to ~150 groups. Include groups for: William/Bill, Robert/Bob, Richard/Dick, James/Jim, John/Jack, Charles/Charlie, Thomas/Tom, Michael/Mike, Edward/Ed/Ted, Henry/Harry, Joseph/Joe, Christopher/Chris, Daniel/Dan, Matthew/Matt, Anthony/Tony, Andrew/Andy, Nicholas/Nick, Donald/Don, Stephen/Steve, Mark/Marc, Patricia/Pat, Elizabeth/Beth/Liz, Jennifer/Jenny, Margaret/Peggy/Meg, Catherine/Kate/Cathy, Susan/Sue, Dorothy/Dot, Rebecca/Becky, Virginia/Ginny, Barbara/Barb, etc.
- Do NOT use `!!` anywhere — use `?: return emptySet()` or `?: return false` for null handling.

**TokenSimilarity.kt:**
- Kotlin `object TokenSimilarity` (per user decision — not a Spring bean)
- `fun overlap(a: String, b: String): Double` — computes overlap coefficient: `|A intersect B| / min(|A|, |B|)`
- Private `tokenize(value: String): Set<String>` — splits on `\s+`, lowercases, filters empty tokens, returns Set
- Returns 0.0 if either token set is empty
- Returns 1.0 for identical non-empty token sets
- Returns 1.0 for subsets (e.g. "John" vs "John Smith") — this is the advantage of overlap coefficient over Jaccard
