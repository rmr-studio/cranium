    Three changes to IdentityMatchCandidateService:

    **1. Add EMAIL branch in findCandidates() loop** — after the existing NAME nickname block (around line 96), add:
    ```kotlin
    if (signalType == MatchSignalType.EMAIL) {
        val domain = EmailMatcher.extractDomain(normalizedValue)
        if (domain != null && !EmailMatcher.isFreeEmailDomain(domain)) {
            val emailDomainCandidates = findEmailDomainCandidates(
                triggerEntityId, workspaceId, normalizedValue, domain, signalType
            )
            allCandidates.addAll(emailDomainCandidates)
        }
    }
    ```
    The free-domain guard lives HERE, not inside findEmailDomainCandidates. This keeps the private method single-purpose.

    **2. Add findEmailDomainCandidates() private method** — follows findNicknameCandidates() pattern exactly:
    - Parameters: triggerEntityId, workspaceId, normalizedEmail, domain, signalType (all needed for SQL + Kotlin filtering)
    - SQL: Select candidate_entity_id, candidate_attribute_id, candidate_value, candidate_signal_type from entity_attributes JOIN entity_type_semantic_metadata. WHERE clause filters by workspace, excludes trigger entity, checks `deleted = false`, and matches domain using `substring(ea.value->>'value' from '@(.+)$') = :domain`. LIMIT $CANDIDATE_LIMIT.
    - Note: SQL does NOT compute similarity — it only fetches same-domain candidates. Similarity is computed Kotlin-side.
    - After SQL execution, for each candidate row:
      a. Extract local part from candidate value via `EmailMatcher.extractLocal(candidateValue)`
      b. Extract local part from trigger via `EmailMatcher.extractLocal(normalizedEmail)`
      c. If either is null, skip this candidate (malformed email)
      d. Compute overlap = `EmailMatcher.localPartSimilarity(triggerLocal, candidateLocal)`
      e. If overlap < 0.5, skip this candidate (below threshold)
      f. Otherwise create CandidateMatch with similarityScore = overlap, matchSource = MatchSource.EMAIL_DOMAIN
    - Add `@Suppress("UNCHECKED_CAST")` annotation (same as other private query methods)
    - Add KDoc explaining the two-phase approach: SQL filters by domain, Kotlin filters by local-part overlap

    **3. Update mergeCandidates() tiebreaker** — change the existing:
    ```kotlin
    .thenBy { if (it.matchSource == MatchSource.NICKNAME) 1 else 0 }
    ```
    to:
    ```kotlin
    .thenBy {
        when (it.matchSource) {
            MatchSource.NICKNAME, MatchSource.EMAIL_DOMAIN -> 1
            else -> 0
        }
    }
    ```
    This prefers EMAIL_DOMAIN over TRIGRAM when deduplicating same (entityId, attributeId) pairs, following the NICKNAME precedent.

    Add import for EmailMatcher (same package, should auto-resolve).

    Best choice for clear test semantics: `j.smith@acme.com` and `john.smith@acme.com` — overlap coefficient = |{"smith"}| / min(2, 2) = 1/2 = 0.5 exactly at threshold.
